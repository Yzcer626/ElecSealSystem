# 电子印章系统 (ElecSealSystem)

[![C++](https://img.shields.io/badge/C++-11-blue.svg)](https://isocpp.org/)[![CMake](https://img.shields.io/badge/CMake-3.10+-green.svg)](https://cmake.org/)[![Platform](https://img.shields.io/badge/Platform-Windows-lightgrey.svg)](https://www.microsoft.com/windows/)

一个基于 C++ 的电子印章生成系统，根据输入的汉字从隶书字体文件生成对应的 BMP 图像。支持多种印章类型、自定义颜色设置和交互式操作界面。

## 功能特性

- **多种印章类型**：
  - 单字印章模式：每个汉字生成独立的 BMP 图像
  - 四字印章模式：四个汉字排列成 2×2 印章（右上→右下→左上→左下布局）
  - 多字印章模式：支持横向或竖向排列

- **颜色自定义**：
  - 12 种预设颜色（红色、蓝色、绿色、黑色、白色、黄色、紫色、青色、橙色、深红色、深蓝色、深绿色）
  - RGB 颜色值输入：支持十六进制（00-FF）精确设置前景色和背景色
  - 实时颜色预览

- **字体支持**：
  - 使用 GB2312 编码的隶书 56×56 点阵字体
  - 支持 7478 个常用汉字

- **图像输出**：
  - 生成 24 位真彩色 BMP 图像
  - 智能文件名管理，避免文件覆盖
  - 输出文件名使用 GB2312 编码的十六进制表示

- **用户界面**：
  - 美观的交互式命令行界面
  - 中文菜单提示
  - 印章生成预览

- **CMake**: 3.10 或更高版本
- **C++ 标准**: C++11

## 快速开始

### 1. 克隆仓库

```bash
git clone https://github.com/yourusername/ElecSealSystem.git
cd ElecSealSystem
```

### 2. 构建项目


## 使用示例

### 单字印章模式

输入汉字如"生日快乐"，程序将为每个汉字生成独立的 BMP 图像：
- `bmp/UC9FA.bmp` (生)
- `bmp/UBFD5.bmp` (日)
- `bmp/UA5F6.bmp` (快)
- `bmp/UD6C0.bmp` (乐)

### 四字印章模式

输入四个汉字，按照"右上→右下→左上→左下"的顺序排列：
- 输入：春夏秋冬
- 布局：左上[秋] 右上[春]
         左下[冬] 右下[夏]

## 技术细节

### 字符编码处理

本项目涉及多种字符编码的转换：

1. **输入**: 使用 Windows 宽字符 API 获取 UTF-8 编码的输入
2. **转换**: UTF-8 → UTF-16 → GBK/GB2312
3. **查找**: 在字体文件中搜索对应的 GB2312 编码
4. **输出**: 使用 GB2312 编码的十六进制作为文件名

### BMP 图像格式

- **格式**: 24 位真彩色 BMP
- **尺寸**: 根据印章类型可变（单字 56×56，四字 112×112）
- **存储**: 从下到上扫描行
- **对齐**: 每行 4 字节对齐

### 字体文件格式

字体文件 `LiShu56.txt` 包含 GB2312 字符集的隶书 56×56 点阵数据：

```
CurCode: XXXX       # 编码标识
width = 7           # 宽度（7组×8位=56像素）
________,________,  # 56行点阵数据
________,________,
...
________,________,

                    # 空行分隔
```

## 常见问题

### Q: 生成的图像无法查看？

A: 确保已安装支持 BMP 格式的图像查看器。Windows 自带的照片应用即可查看。

### Q: 某些汉字无法生成？

A: 字体文件只包含 GB2312 字符集的 7478 个常用汉字，部分生僻字可能无法找到。

### Q: 如何获取颜色值？

A: 可以使用 Windows 自带的截图工具（Win+Shift+S）的取色器功能，获取光标处的颜色值。

### Q: 如何修改默认颜色？

A: 在颜色设置菜单中选择"恢复默认颜色"，或手动设置前景色为红色（FF0000），背景色为白色（FFFFFF）。
